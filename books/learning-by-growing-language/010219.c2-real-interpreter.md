こんなコードを見ても、インタプリタ作った感じがしない？まあ、これって言っちゃえば Python のコードですしね。

```py
    toil.walk(r"""
        def factorial_rec(n) do
            if n == 0 then 1 else n * factorial_rec(n - 1) end
        end
    """)
```

やっぱり、ソースコードをエディタで書いてそれを実行しないと本当の言語処理系って感じがしませんね。

やりましょう。

ついでに REPL も作りましょう。超簡単な奴ですが。REPL というのは Python で言えばこれのことです。

```py
>>> 2 + 3
5
```

`2 + 3` という入力を Read して Eval して結果を Print する Loop だから REPL と呼ばれます。この説明はもう耳にタコができてるかもしれませんが、今は Eval が何をしているかわかってる分解像度が上がってるはずです。Read は入力＋字句解析＋構文解析だと思ってください。

今回の修正は `__main__` セクションだけです。処理系本体、つまり `Interpreter` をはじめ各種クラスはそのままです。

```py
 if __name__ == "__main__":
+    import sys
 
     toil = Interpreter()
 
+    def repl():
+        while True:
+            print("\nInput source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):")
+            if (src := sys.stdin.read()) == "":
+                exit(0)
+            try:
+                expr = toil.ast(src)
+                print("AST:", expr, sep="\n")
+                print("Output:")
+                result = toil.eval(expr)
+                print("Result:", result, sep="\n")
+            except AssertionError as e:
+                print("Error:", e, sep="\n")
+
+    def from_file(filename):
+        with open(filename, "r") as f: result = toil.walk(f.read())
+        exit(result if isinstance(result, int) else 255)
+
+    match sys.argv:
+        case [_]: pass
+        case [_, "--repl"]: repl()
+        case [_, filename]: from_file(filename)
+        case _: assert False, f"Invalid command line: {sys.argv}"
 
     # Example

     ...
```

解説より先に動かしてみましょうか。

まずは REPL から。`python toil.py --repl` で起動すると入力待ちになります。

```
$ python toil.py --repl

Input source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):

```

`$` は OS 等の環境によって変わります。

ここで Toil のソースコードを入力し、Enter を押して最下行に何も入力してない状態で、

Linux/Mac:  Ctrl+D を押してください。
Windows: Ctrl+Z を押してさらに Enter を押してください。

これらはそれぞれの OS で EOF (End Of File) を表す入力です。ちょっと変わった入力方法ですが、複数行の入力を簡単に済ませるためです。複数行のテキストをどこかでコピーしてここに貼ることも普通にできます（最後なにもない行で Ctrl+D するのは忘れずに）。

以下の例では Enter や Ctrl-D の入力も書いていますが、実際に画面に表示されるわけではありませんので注意してください。

```
Input source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):
2 + 3 <Enter>
<Ctrl-D>
AST:
('add', [2, 3])
Output:
Result:
5
```

`2 + 3` を入力（Read）すると、評価（Eval）されて結果が `5` と表示（Print）されます。AST も表示しているのは何かの役に立つかな、くらいのものです。

```
Input source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):
print(2 + 3) <Enter>
<Ctrl-D>
AST:
('print', [('add', [2, 3])])
Output:
5
Result:
None
```

結果を表示したら入力待ちに戻りますのでまた入力して実行させることができます。`print` の結果は `Output:` の下に表示されます。`Result:` には `print()` の戻り値である `None` が表示されています。

```
Input source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):
a <Enter>
<Ctrl-D>
AST:
a
Output:
Error:
Undefined variable @ val(): a
```

エラーの場合。

```
Input source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):
if 2 == 2 then <Enter>
    3 <Enter>
else <Enter>
    4 <Enter>
end <Enter>
<Ctrl-D>
AST:
('if', [('equal', [2, 2]), 3, 4])
Output:
Result:
3
```

複数行入力しても大丈夫。Python じゃないのでこれくらいなら 1 行に入れちゃってもいいんですが。

```
Input source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):
<Ctrl-D>
$
```

なにも入力しないで Ctrl-D すると REPL を終了します。

Toil が自分でチェックしたエラー（その場合は AssertionError が上がる）のときは REPL の実行は続きますが、それ以外でエラーが発生した場合はその場で終了してしまいます。すべて自分でチェックしてそういうエラーが出ないようにするのが理想ですが、Toil はそこにはこだわっていません。

```
Input source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):
1/0
AST:
('div', [1, 0])
Output:
Traceback (most recent call last):
...
ZeroDivisionError: integer division or modulo by zero
$
```

次はファイルを実行しましょう。まず、ソースファイルを作ります。

```text:gcd.toil
gcd := func a, b do
    if a == 0 then b else gcd(b % a, a) end
end;
print(gcd(24, 36));
0
```

最後に評価した式の値を終了ステータスにするようになっているので最後に 0 を書いています。`exit(0)` みたいなものです。といってもそこでプログラムを exit するわけではありませんが。

実行するときは `python toil.py gcd.toil` です。終了ステータスは Linux/Mac なら `echo $?` で、Windows なら `echo %errorlevel%` で確認することができます。

```text:Linux/Mac
$ python toil.py gcd.toil 
12
$ echo $?
0
```

```text:Windows
>python toil.py gcd.toil
12
>echo %errorlevel%
0
```

最後の `0` がなければ最後に評価される式は `print()` で、これは `None` を返しますが、数以外の結果になったときは終了ステータスを 255 にしています。

```text:Linux/Mac
$ python toil.py gcd.toil 
12
$ echo $?
255
```

```text:Windows
>python toil.py gcd.toil
12
>echo %errorlevel%
255
```

どうでしょう？本当にインタプリタが動いてる、って感じがしてきたでしょうか？

ではパーツごとに解説していきます。

```py
    import sys
```

Toil 史上初の `import` です。つまりこれまで混じりっ気のない純粋な Python だけで他の何ものの手も借りず処理系を書いてきたことになります[^not-impure]。すばらしい。

[^not-impure]: 別に `sys` が混じりっ気のある不純な Python だと言いたいわけではありませんよ。

```py
    match sys.argv:
        case [_]: pass
        case [_, "--repl"]: repl()
        case [_, filename]: from_file(filename)
        case _: assert False, f"Invalid command line: {sys.argv}"
```

簡単なコマンドライン解析です。

たとえば `python toil.py sample.toil` として Toil を起動すると、 `sys.argv` は `["toil.py", "sample.toil"]` になります。

オプションがついてなければここはスルーして `# Example` 以下のコードに進みます。
コマンドライン引数がひとつで、それが `--repl` であれば REPL を起動します。
それ以外でコマンドライン引数がひとつならば、それをファイル名と見て実行します。
それ以外の形ならエラーとして終了します。

```py
    def repl():
        while True:
            print("\nInput source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):")
            if (src := sys.stdin.read()) == "":
                exit(0)
            try:
                expr = toil.ast(src)
                print("AST:", expr, sep="\n")
                print("Output:")
                result = toil.eval(expr)
                print("Result:", result, sep="\n")
            except AssertionError as e:
                print("Error:", e, sep="\n")
```

REPL を実行するメソッドです。

`sys.stdin.read()` は標準入力（通常はキーボード）からの入力を受け取ります。Enter では入力は終わらず、EOF（標準入力の場合は Ctrl+D や Ctrl+Z）まで複数行でも全部まとめて受け取ります。受け取った文字列が空であれば終了します。あとは順番に `ast()`、`eval()` を呼んで入力されたソースコードを実行します。`AssertionError` が発生したらメッセージを表示します。これを繰り返します。

```py
    def from_file(filename):
        with open(filename, "r") as f: result = toil.walk(f.read())
        exit(result if isinstance(result, int) else 255)
```

指定されたファイルを読み込み、`walk()` で実行します。`walk()` の値が数ならばそのまま、数でなければ 255 をステータスとしてプログラムを終了します。

今度こそインタプリタ完成です！

## 第 2 章のまとめ

第 1 章の `eval()` はそのままに、字句解析と構文解析を追加して、人間にわかりやすいソースコードを実行できるようにしました。ここまで書ければ「ちょっとしたプログラミング言語書いたことあります」って言えます。

プログラムは 300 行程度になりましたが、それでも思っていたよりずっと短いのではないでしょうか？まだエディタの画面で少しスクロールすれば全体が見えるレベルです。

字句解析・構文解析と評価が完全に分離しているアーキテクチャーにも注目してください。このことにより、字句解析・構文解析器と評価器をそっくり別モノと入れ替えることができます。字句解析・構文解析器を変更すれば、評価器はそのままに、見かけはまったく別の言語にすることもできます。

人間に近い字句解析・構文解析器の部分をフロントエンド、反対に評価器のことをバックエンドとも言います。

次章ではバックエンドを中間コード＋仮想マシンを使った、まったく違う仕組みのものに入れ替えます。

## コラム：シバン

Linux・Mac の環境では、`toil.py` の先頭に以下の行を追加し、

```
#! /usr/bin/env python3
```

`toil.py` に実行権限をつけると、python と書かずに実行できます。`#!` で始まる行をシバン（Shebang）と言います。

```
$ chmod a+x toil.py
$ ./toil.py gcd.toil
12
$
```

さらに、`toil.py` を toil という名前でパスが通っているところに置くと本物感が増します。

```
$ sudo cp toil.py /usr/local/bin/toil
$ toil gcd.toil
12
```
