title: "2.16 func-do-end"
---

ユーザ定義関数を書けるようにします。
ソースコード上では `func <引数>, <引数>, ... do <本体> end` と書くことにします。AST は `("func", [[<引数>, <引数>, ...], <本体>])` に変換します。`<引数>` はいくつあるかわからないので配列に入れるようにします。`<引数>` は 0 個でもかまいません。

`eval()` の時はクロージャだのなんだのとややこしい話がありましたがここではいつものように淡々と文字列から AST へ変換するだけです。

```diff py
 class Parser:
     ...
 
     def _primary(self):
         match self._current_token():
             ...
             case "(": return self._group()
+            case "func": return self._func()
             case "scope": return self._scope()
             ...

     ...

     def _group(self):
         ...
 
+    def _func(self):
+        self._current_and_advance()
+        params = self._comma_separated_exprs("do")
+        self._consume("do")
+        body_expr = self._expression()
+        self._consume("end")
+        return ("func", [params, body_expr])

     def _scope(self):
         ...
```

`<引数>, <引数>, ...` を読むときに `_comma_separated_exprs()` を使ってるところがちょっとだけ新しいくらいでしょうか。`<引数>` の並びは `"do"` で終わるので `_comma_separated_exprs()` に `"do"` を渡しています。

実行例です。

```py
    print("Function:")

    print(toil.ast(r""" func do 2 end """)) # -> ('func', [[], 2])
    print(toil.walk(r""" func do 2 end """))
    # -> ('closure', [[], 2, <__main__.Environment object at ...>])
    print(toil.walk(r""" func do 2 end () """)) # -> 2

    print(toil.ast(r""" func a do a + 2 end """))
    # -> ('func', [['a'], ('add', ['a', 2])])
    print(toil.walk(r""" func a do a + 2 end """))
    # -> ('closure', [['a'], ('add', ['a', 2]), <__main__.Environment ...>])
    print(toil.walk(r""" func a do a + 2 end (3) """)) # -> 5

    print(toil.ast(r""" func a, b do a + b end """))
    # -> ('func', [['a', 'b'], ('add', ['a', 'b'])])
    print(toil.walk(r""" func a, b do a + b end """))
    # -> ('closure', [['a', 'b'], ('add', ['a', 'b']), <__main__.Environment ...>])
    print(toil.walk(r""" func a, b do a + b end (2, 3) """)) # -> 5
```

引数が 0 個、1 個、2 個のときにちゃんと処理できているので 3 個以上でもきっと大丈夫。

```py
    # toil.walk(r""" func a, b a + b end """) # -> Expected do
    # toil.walk(r""" func a, b do end """) # -> Expected end
    # toil.walk(r""" func a, b do a + b """) # -> Expected end
    # toil.walk(r""" func a, do a + b end """) # -> Expected do
    # toil.walk(r""" func , b do a + b end """) # -> Invalid token

    print(toil.walk(r"""
        twice := func f, x do f(f(x)) end;
        double := func x do x * 2 end;
        twice(double, 3)
    """)) # -> 12

    print(toil.walk(r"""
        a := 2;
        f := func do a end;
        g := func do a := 3; f() end;
        g()
    """)) # -> 2
```

引数に関数を入れて渡せてるとか、静的スコープになってるとかのテストですが、そのあたりのお仕事は `eval()` がやってるのでなんだっけこれと思ったら 1 章の実行例や説明と見比べてみてください。


```py
    print(toil.walk(r"""
        func a do func b do a + b end end (2)(3)
    """)) # -> 5
```

「2.11 関数呼び出し」で「現時点ではまだ意味のあるコードは書けませんが」と AST だけ作ってみた関数呼び出しの連鎖を実際に動かしています。

```py
    print(toil.walk(r"""
        f := func a do
            g := func b do a + b end;
            g
        end;
        f(2)(3)
    """)) # -> 5
```

上の実行例と同じことを少し説明的に書いたものです。`f(2)` を呼ぶと、`a` が `2` である環境の中で `func b do a + b end` が評価されるので、`g` は `func b do 2 + b end` になり、それが `f()` から返されます。`f(2)(3)` は `g(3)` ということになり、`func b do 2 + b end` の `b` に `3` が渡されて最終的には `5` となります。

おめでとうございます！これで Toil の文法が `eval()` の機能をすべてカバーしました！

でも、あとひとつ機能を追加します。`eval()` は変更せずに。構文解析だけで機能を追加することもできる（こともある）のです。
