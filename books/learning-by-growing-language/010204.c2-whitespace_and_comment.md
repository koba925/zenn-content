---
title: "2.4 空白とコメント"
---

これからインタプリタに機能を足していきますがまずは小手調べとして、字句解析時点でソースコードの空白やコメントをスキップするようにします。字句解析で空白やコメントを捨ててしまうことで、構文解析以降はそれらのことを考えなくてよくなり、本来の仕事に集中できます。字句解析→構文解析→評価と段階を分けて作る効果がこういうところに出てきます。

```diff py
 class Scanner:
     ...
 
     def tokenize(self):
         while True:
+            while self._current_char().isspace(): self._advance()
             ...
```

トークンを読み始める前に、空白以外の文字が出てくるまで現在位置を進めます。`isspace()` はタブ（`"\t"`）や改行（`"\n"`）も空白とみなすので、それらも無視されます。

続いて、

```diff py
             while self._current_char().isspace(): self._advance()

+            if self._current_char() == "#":
+                while self._current_char() not in ("\n", "$EOF"):
+                    self._advance()
+                continue

             self._start_pos = self._current_pos
             ...
```

`"#"` が出てきたらそこから行末またはソースコード末尾までをコメントとみなし、読み飛ばして、ループの先頭（空白を読み飛ばすところ）から繰り返します。

実行例です。

```py
    print("Whitespaces:")
    print(toil.walk(r""" 2 """))  # -> 2
    print(toil.walk(r"""
        2
    """))  # ->  2

    # print(toil.walk(r""" """))  # -> Invalid token
    # print(toil.walk(r""" 2 34 """))  # -> Extra token

    print("Comment:")

    print(toil.walk(r""" 2 # Comment """))  # -> 2
    print(toil.walk(r"""
        # Comment
        2
        # Comment
    """))  # -> 2
```

トリプルクォートを使うことで、複数行のコードも書けています。なんかちょっとソースコードに見えてきましたね。
