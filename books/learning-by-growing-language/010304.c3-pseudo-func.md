第 1 章でもやったように簡易版の関数を作ります。簡易版と言っても、仮想マシンに命令を追加してスタックから値を取り出して処理するという正当な方法です。あとで `_builtins()` の関数を使う方式に置き換えるときに削除してしまいますが、むしろ基本はこっちなのでしっかり理解しておいてください。スタックマシンの原理ともいえる部分です。

仮想マシンから修正します。

```diff py
 class VM:
     ...
 
     def execute(self):
         while (inst := self._code[self._ip]) != ("halt",):
             self._ip += 1
             match inst:
                 case ("const", val): self._stack.append(val)
+                case ("print",):
+                    print(self._stack.pop()); self._stack.append(None)
+                case ("add",):
+                    r = self._stack.pop(); l = self._stack.pop()
+                    self._stack.append(l + r)
+                case ("mul",):
+                    r = self._stack.pop(); l = self._stack.pop()
+                    self._stack.append(l * r)
+                case ("equal",):
+                    r = self._stack.pop(); l = self._stack.pop()
+                    self._stack.append(l == r)
+                case ("less",):
+                    r = self._stack.pop(); l = self._stack.pop()
+                    self._stack.append(l < r)
                 case _:
                     assert False, f"Invalid instruction @ execute(): {inst}"
         assert len(self._stack) == 1, \
            f"Invalid stack state @ execute(): {self._stack}"
         return self._stack.pop()
```

`print`、`add`、`mul`、`equal`、`less` という５つのオペコードを追加しています。これらの命令自体にはオペランドがないことに注意してください。では何を `print` したり `add` するのかというと、スタックに入っている値です。

スタックマシンのお約束としてスタックに値を積んで返すというのがあるよ、という話をしましたが、それを含めてスタックマシンの動作をまとめるとこんな感じです。

* 命令ごとに定まった適切な数の入力をスタックに積んでおく
* 命令の処理では積まれた値を決まった数だけスタックから取り出す
* 取り出した値を入力としてなんらかの仕事を行う
* 結果をスタックに積んで出力とする
* これを繰り返す

`add` では `r = self._stack.pop(); l = self._stack.pop()` でスタックからふたつ値を取り出し、`self._stack.append(l + r)` で `l + r` をスタックに積んでいます。スタックに残った `l + r` が `add` の値となります。

`print(self._stack.pop())` は計算するわけではありませんが考え方は同様で、スタックから値をひとつ取り出してそれを `print` したあと `self._stack.append(None)` でスタックに `None` を積みます。これが `print` の値となります。この `print` 命令はひとつしか引数をとらない `print()` のように動くわけです。

ほかの命令も同様。

次はコンパイラです。

まず、`_code` に命令を追加する `_emit()` ヘルパーメソッドを導入します。

```diff py
 class Compiler:
     ...
  
+    def _emit(self, *inst):
+        self._code.append(inst)

 
 class VM:
     ...
```

`_emit` を使うと `self._code.append(("halt",))` が `self._emit("halt")`、`self._code.append(("const", expr))` が `self._emit("const", expr)` と書けて見た目がかなりすっきりします。`*inst` のように関数定義で引数に `*` をつけると、指定された引数がタプルに入って渡されるという Python の仕組みを使っています。Emit というのは放射する・放出するといった意味なのですが、どういうわけかこういう文脈ではよく出てきます。


```diff py
class Compiler:
     def __init__(self, expr):
         self._expr = expr
         self._code = []
 
     def compile(self):
         self._expression(self._expr)
-        self._code.append(("halt",))
+        self._emit("halt")
         return self._code
 
     def _expression(self, expr):
         match expr:
-            case None | bool() | int(): self._code.append(("const", expr))
+            case None | bool() | int(): self._emit("const", expr)
+            case (op, [expr]):
+                self._expression(expr)
+                self._emit(op)
+            case (op, [left_expr, right_expr]):
+                self._expression(left_expr)
+                self._expression(right_expr)
+                self._emit(op)
             case _: assert False, f"Unsupported expression @ compile(): {expr}"
 
     def _emit(self, *inst):
         ...
```

`case None | bool() | int():` のところは `_emit()` で書き換えただけです。

`case (op, [expr]):` は 1 引数の関数呼び出しの形です。引数がひとつなので、スタックにひとつ引数を積んでから命令を実行します。`self._expression(expr)` で `expr` を実行してその値をスタックに積むコードを出力し、その後 `op` を `emit()` することでそれを実現します。

`case (op, [left_expr, right_expr]):` は 2 引数の形で、考え方は 1 引数のときと同じですが注意してほしいのはスタックに積む順番とスタックから取り出す順番が逆転していることです。

以下のコードでは左の式の値をスタックに積んだ後、右の式の値をスタックに積むコードを出力します。

```py
                self._expression(left_expr)
                self._expression(right_expr)
```

一方、`add` や `mul` を実行するときは以下のように、右の値を取り出してから左の値を取り出しています。

```
                    r = self._stack.pop(); l = self._stack.pop()
```

スタックに「積む」「取り出す」動作を考えるとそれが自然ということがわかると思います。足し算やかけ算では順番が入れ替わっても計算結果は変わりませんが、引き算や割り算、その他多くの場合は順番が変わってしまうと結果に影響が出ますので注意してください。

スタックマシンの動作の「スタックに積んでおく」というステップは、スタックマシンというよりもコンパイラの仕事で、そういう AST を出力してやる、ということになります。もっとも単純なのは `const` で値を積むことですが、たとえば `2` と `3` をスタックに積んで `add` してやることによって、結果的には `5` をスタックに積んだことになります。その後 `4` をスタックに積んで `mul` してやると `20` がスタックに積まれ、`(2 + 3) * 4` を計算したことになります。このようにスタックを使っていくらでも計算を連鎖させることができます。

スタックの伸び縮みを思い浮かべながら実行例を見てみましょう。

```py
    print("Pseudo functions:")

    print(toil.ast(r""" 2 + 3 """))
    # -> ('add', [2, 3])
    print_code(toil.code(r""" 2 + 3 """))
    # ->  0: ('const', 2)
    # ->  1: ('const', 3)
    # ->  2: ('add',)
    # ->  3: ('halt',)
    print(toil.run(r""" 2 + 3 """)) # -> 5
```

`2` を積み `3` を積んだ後、`3` を取り出し `2` を取り出して足し、結果である `5` を積んで終わります。最後にスタックに残った `5` が `2 + 3` の値となります。

```py
    print(toil.ast(r""" 2 + 3 * 4 """))
    # -> ('add', [2, ('mul', [3, 4])])
    print_code(toil.code(r""" 2 + 3 * 4"""))
    # ->   0: ('const', 2)
    # ->   1: ('const', 3)
    # ->   2: ('const', 4)
    # ->   3: ('mul',)
    # ->   4: ('add',)
    # ->   5: ('halt',)
    print(toil.run(r""" 2 + 3 * 4 """)) # -> 14
```

スタックの状態を追いかけてみましょう。

```
初期状態     -> []
('const', 2) -> [2]
('const', 3) -> [2, 3]
('const', 4) -> [2, 3, 4]
('mul',)     -> [2, 12]
('add',)     -> [14]
```

スタックに残った唯一の値 `14` が `2 + 3 * 4` の結果ということになります。

`[2, 12]` という状態は「`2` と、`3 * 4` の結果をスタックに積んだ」と言える状態になっていて、その状態で `mul` を実行することが「`2` と `3 * 4` をかけ算した」とみなせることに注意してください。


```py
    print(toil.ast(r""" (2 + 3) * 4 """))
    # -> ('mul', [('add', [2, 3]), 4])
    print_code(toil.code(r""" (2 + 3) * 4"""))
    # ->   0: ('const', 2)
    # ->   1: ('const', 3)
    # ->   2: ('add',)
    # ->   3: ('const', 4)
    # ->   4: ('mul',)
    # ->   5: ('halt',)
    print(toil.run(r""" (2 + 3) * 4 """)) # -> 20
```

同じようにやってみます。

```
初期状態     -> []
('const', 2) -> [2]
('const', 3) -> [2, 3]
('add',)     -> [5]
('const', 4) -> [5, 4]
('mul',)     -> [20]
```

`add` を実行した時点で `2 + 3` をスタックに積んだ状態になっており、さらに `4` を `mul` することで `(2 + 3) * 4` が計算できたことになっています。

```py
    print(toil.run(r""" 2 + 3 == 2 * 3 """)) # -> False
```

どういうコードが出力されるか想像できますか？`2 + 3` をスタックに積み、 `2 * 3` をスタックに積み、`equal` を実行するようなコードです。答え合わせは `ast()` を呼ぶか REPL で。

あとはノーヒントで大丈夫でしょう。`print` はひとつだけしかスタックから値を取り出さないことくらいでしょうか。

```py
    print(toil.run(r""" 2 + 3 < 2 * 3 """)) # -> True
    print(toil.run(r""" print(2 + 3) """)) # -> 5\nNone
```

これだけでもうスタックマシンの根本は理解できたことになります。命令を追加すれば立派な電卓になります[^notation]。

[^notation]: Toil では `2 + 3 * 4` のような式が解釈できる構文解析器を作っていますが、`2 3 4 * +` のようにスタックマシン用のコードと同じ順番で値や演算子を書くようにするとうんと簡単なプログラムで電卓を作ることができ、インタプリタ作りの最初のテーマになることもあります。`2 3 4 * +` のように、対象の値を先に書き、演算子をあとに書く書き方を逆ポーランド記法と言います。当然ポーランド記法というのもあって、これは最初に演算子を書き、その対象をあとに書くというもので、`('mul', [('add', [2, 3]), 4])` といった Toil の AST はまさしくポーランド記法の例となっています（よく例に上がるのは LISP の S 式です）。`2 + 3 * 4` のように演算子を間に書く書き方は中置記法と言います。

スタックの動きがイメージできるようになったら次へ進みましょう！
