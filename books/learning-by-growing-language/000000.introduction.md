本書は Toil [^toil]というプログラミング言語を作りながら、プログラミング言語処理系の原理を学ぶための本です。ソースコードを処理して実行するプログラムをプログラミング言語処理系と呼ぶのですが、ちょっと長いので本書では言語処理系・処理系などと呼ぶことにします。

[^toil]: 趣味で作るおもちゃのような言語をトイ言語と呼びます。Toil という名前は TOI Language から来ています。

プログラミング言語ってどうやって動いてるんだろう？と興味を持っている人に向けて書きました。Python をすこし覚えて、配列[^list]（`[2, 3, 4, ...]`）や辞書（`{"a": 2, "b": 3, ...}`）を使ったり自分でクラスを作ったことがある、くらいの人が読んでいるところを思い浮かべて書いています。

[^list]: Python 用語では「リスト」ですが、本書では他の言語でも使われる用語である「配列」を使っていきます。

難しいかなと思ったところは簡単に説明しながら進めますし、もともとそれほどややこしい機能は使いませんので安心して読み進めていただけると思います。知ってる方は適宜読み飛ばしていただければ。

## 本書の特徴

本書ではインタプリタ型の処理系をごく小さなプログラムから始めてインクリメンタルに機能を付け足しては動かしてみることを繰り返すことで学習していきます。

「インタプリタ」というのは、すごくざっくり言うとソースコードを読んですぐ実行する処理系のことです。対となる言葉は「コンパイラ」で、こちらはいったんソースコードを他の形式に変換しておき、実行時にはその変換したものを使う処理系を指します。このあたりは細かいことに踏み込むと長くなってしまうのでこれくらいで。

:::message
この本は理論や用語にはあまり深く踏み込まず、だいたいこれくらいのざっくりさで雰囲気をつかむ程度の説明でいきます。ただ、用語についてはその後の学習につながるよう、一般的な用語を使います。初めて出てくる用語はかんたんに説明するようにしますが、すこし難しく聞こえる言葉もあるかもしれません。あまり気にせず先へ進んでみてください。書いて動かしてを繰り返しているうちにそういうことだったか、とわかるようになると思います。
:::

第 1 部の第 1 章が終わると 100 行程度のプログラムでもうこんなソースコード（？）が動かせるようになります。これは階乗（1 からある数まですべてかけ算したもの）を求める関数を定義しています。今は理解できなくても問題ありませんが、`define` とか `while` とかなんとなくプログラムっぽいことをしてる雰囲気は感じられるのではないでしょうか？

```py
    ("define", ["factorial_iter", ("func", [["n"], ("seq", [
        ("define", ["result", 1]),
        ("while", [("greater", ["n", 0]), ("seq", [
            ("assign", ["result", ("mul", ["result", "n"])]),
            ("assign", ["n", ("sub", ["n", 1])]),
        ])]),
        "result"
    ])])])
```

章はさらに節に分かれていて、節ごとに機能を追加し、動かしながら進みます。

第 2 章を終わると同じプログラムをこういう風に書けるようになります。これなら「ちょっと違う言語なんだな」くらいの感じで読めるのではないでしょうか。上にあげた第 1 章のプログラムもこれと見比べてみるとああそういうことか、とわかるかもしれません。

```
    def factorial_iter(n) do
        result := 1;
        while n > 0 do
            result = result * n;
            n = n - 1
        end;
        result
    end
```

第 3 章では第 1 章とは異なるやりかたでプログラムを動かします。ソースコードを中間コードに翻訳するコンパイラと、その中間コードを実行する仮想マシンを組み合わせてプログラムを動かす、というものです。この本はインタプリタに関する本ですが、中間コードへとは言えコンパイルするのでコンパイラの処理の片鱗を見ることもできます。

## 全体の構成と読み進め方

本書はインクリメンタルに開発を進めていくということもあって、順番に読むことを想定しています。ある程度わかっているから興味のあるところだけ読む、という読み方ももちろんしていただいてかまいませんが、用語や Python の機能については初出時に説明するようにしていたり、コードを差分形式で書いたりしているため、若干読みづらくなる可能性があります。

全体の構成は以下のようになっています。

* はじめに
* 第 1 部　最小のインタプリタ
  機能を最小限に絞り、ツリーウォーク式インタプリタ・中間コード式インタプリタというふたつの方式で小さなインタプリタを作ります。
  * 第 1 章　最小の評価器 (100 行)
    ツリーウォーク式インタプリタのコア部分を作り、アルゴリズムを実行できるようにします。  
  * 第 2 章　最小の字句解析・構文解析器 (300 行)
    テキストのソースコードを読んで、第 1 章の評価器で読めるようにする字句解析・構文解析器を作ります。
  * 第 3 章　最小の中間コード式インタプリタ [^bytecode]  (500 行)
    第 1 章の評価器の代わりに動く、ソースコードを中間コードに翻訳するコンパイラおよびその中間コードを実行する仮想マシンを作ります。
* 第 2 部　ツリーウォーク式インタプリタの拡張（以下未稿）
  ツリーウォーク式インタプリタに機能を追加していきます。第 2 部が終わると機能だけはわりと最近のスクリプト言語に近いレベルになります。
  * 第 4 章　基本のインタプリタ
    配列型・文字列型・タプル型を導入し、制御構造を強化します。ここまでくらいで自作プログラミング入門としては十分な機能と言えます。
  * 第 5 章　辞書
    辞書型を導入します。ドット記法を使ってパイプライン的記述・メソッド的記述ができるようにします。
  * 第 6 章　パターンマッチ
    関数の可変長引数から始めて分割代入・`match` 文・アロー関数などパターンマッチを使った各種機能を導入します。
* 第 3 部　メタサーキュラー
  Toil で Toil 自身を扱う方法を学びます。
  * 第 7 章　メタサーキュラー構造
    `eval()` や `apply()` を導入したり、Toil 言語で Toil 処理系を書いたりします。
* 第 4 部　拡張可能な言語
  Toil で Toil を拡張できるようにします。
  * 第 8 章　ファーストクラスマクロ
    マクロを導入し、処理系自体に手を入れなくても Toil の機能を拡張できるようにします。マクロの原理を学ぶため、まずは簡易な方式(評価中にマクロを展開する)で実装します。
  * 第 9 章　カスタム文法
    マクロの呼び出しを、if や while といった構文と同じようなスタイルで書けるようにする文法カスタマイズ機能を導入します。
  * 第 10 章　事前展開型マクロ
    あらかじめマクロを展開してから評価器に渡す、より本格的なマクロを実装します。
* 第 5 部　中間コード式インタプリタの拡張
  第 2 部・第 3 部で行った機能拡張を、今度は中間コード式インタプリタで行います。
* 第 6 部　ツリーウォーク式インタプリタと中間コード式インタプリタの協調動作
  これまでツリーウォーク式インタプリタと中間コード式インタプリタはほぼ独立に動作していましたが、協調して動作できるようにします。
  * ツリーウォーク式インタプリタの関数から中間コード式インタプリタの関数を呼んだり、その逆をできるようにします。
  * ツリーウォークインタプリタを実行中に関数をコンパイルできるようにします。
  * 最後は JIT モドキを作って完成となります。
* まとめ

[^bytecode]: インタプリタ実装の文脈では「バイトコード」という言葉の方がよくつかわれているかもしれません。本書では中間コードを「バイト」にしないので「中間コード」と呼んでいます。

字句解析・構文解析から始めて評価器あるいは中間コードコンパイラと仮想マシンに進む、という構成になっている書籍が多いのですが、本書は評価器を先に持ってきています。言語処理系がソースコードを処理していく順番に合わせれば字句解析・構文解析が先に来るのは自然なのですが、評価器を作るほうが（「本書の特徴」で挙げたような）プログラムっぽいものが動いたほうがきっと楽しいですよね！

おすすめの読み方は、一度ざっと読んだら自分で書いてみることです。読むだけでもそれなりの理解はできますが、自分で書いてみることにより飛躍的に理解度が上がります。「あれ、ここどうなってたっけな？」と思ったらすぐに書籍を見てかまいません。自分で書いてみるというところが大切で、わかったつもりのところが実はわかってなかった、というところがたくさん見つかるでしょう。それからもう一度書籍のコードを見ることで理解が進みます。

私自身、わかってると思ってたのにまったく書けなくて愕然としたことがあります。

いきなり自分で書くのは難しすぎる、という場合はじっくり読んでから再挑戦したり、いわゆる「写経」をしてみるのも手でしょう。掲載されているコードをひたすら書き写す、というやつです。その後、もう一度自分で書けるかどうかやってみてください。

本書のサンプル通りに書いたら、ちょっと改造してみるというのもよい練習になります。修正したままにして先へ進むとその先で動かなくなってしまうかもしれないので、コピーして別のところでやるとかブランチを切って試すとかしてもとのコードは残しておいてください。

この書籍では、第 1 部を読むだけでインタプリタ処理系の全体像がつかめるようになっています。ざーっと第 1 部の最後まで眺めたら、第 2 章を読んで書いてみる、3 章を読んで書いてみる、のように小刻みに進めてみるとよいと思います。なんなら第 2 章だけ読んで書いてみる、でも。

第 1 部を理解できればもう自分で言語を拡張したり、1 から自分の言語を書いたりすることもできるはずです。第 2 部以降を読んで書いてみるというのはもちろんですが、先に自分で書いてみるのも面白いかと。

## なぜPythonなのか

広く普及していて Python なら読み書きできるという人が多いというのは大きな理由の一つですが、それだけではありません。

インタプリタを作りましょうという本はたいてい C 言語や Java など、高速な部類の言語を使っています。インタプリタの性能はそれを書いた言語の速度に直接影響を受けてしまうためです。残念ながら Python の実行速度は速いとは言えず、本来はインタプリタを書くのには適していません。にもかかわらず本書で Python を使うのは、「原理を学ぶ」には Python が最適だと考えているからです。

Python はアルゴリズムを自然に見やすく書けるだけの機能が十分にそなわっており、かつ型などを書かなくてもよいため、プログラムに表現したいことだけを素直に書くことができます。

アルゴリズムを教えるとき、読みやすさを重視して「疑似コード」と呼ばれる架空の言語で説明するときがあります。Python のプログラムは、ほとんどこの「疑似コード」に近い読みやすさで、かつ実際に動かすことができるため、原理を学ぶには最適の言語と言えます。

## 本書でやらないこと

本書はコンパクトなプログラムで原理を示すことを狙っているので、それ以外のことはいさぎよく切り捨てています。これらについては別の書籍で学ぶなり自力でがんばってみるなりしてください。

* エラー処理
  Toil では、エラーを見つけたら即停止で、行番号も教えてくれません。
  実用的な処理系では、1 回の実行（コンパイラなら 1 回のコンパイル）でできるだけ多くのエラーを見つけ、できるだけわかりやすく表示することが大切です。
* 堅牢性
  Toil はたとえば `1 / 0` など実行すればエラーになる式もチェックせずに実行するので、Toil のエラーではなく Python のエラーで止まります。 正しい入力に対しては正しく動くという点については注意するけれども、不正な入力について適切なエラーになるかどうかは気にしていない、とも言えます。  Python のエラーで止まってしまうかもしれないし、想定外の動きをしてしまうかもしれません。実用的な処理系では、自分でチェックして適切にエラー処理を行うことが大切です。
* 性能
  先ほど書いた通り、Python で書いているので性能は推して知るべしです。高速な処理系にしたければ高速な言語で。
* メモリ管理
  変数、配列などを記憶するメモリをどう獲得してどう配置し、いらなくなったメモリをどう破棄するか[^gc]というのも大事な話なのですが本書では扱いません。そのあたりは Python にまかせています。というか Python でそのあたりを書くのは少々無理があります。できないことはありませんが・・・

[^gc]: ガベージコレクション（Garbage Collection: GC）と呼ばれます。できることならやりたい話題なんですが処理系を根底から書き直すことになるので見送っています。もともと性能は度外視しているとはいえ、Python ではさすがに処理速度的にも厳しいかなあ・・・

```
$ python --version
Python 3.12.13
```
