title: "2.11 関数呼び出し"
---

関数を呼び出せるようにします。なんか難しそうな気がするかもしれませんが字句解析・構文解析はあくまでソースコードを AST に変換するのが仕事で、難しいところは評価器がやってくれてますのでそう大変でもありません。ただ今までやってきたこととは構文が違った形をしているのでそのへんは新しいパターンが必要です。

関数呼び出しは見慣れた `<関数>(<式>, <式>, ...)` という形とします。具体例としては `add(2, 3)` のような形です。`<式>` は 0 個でもかまいません。`<関数>` は `add` のような関数名だけでなく、式が書けるものとします。`<関数式>` と書いたほうがいいかもしれませんね。AST は `[<関数>, [<式>, <式>, ...]]` です。おなじみの形。

`+` や `*` みたいな演算子とはちょっと形が違うな、どう書くんだろうな・・・と思うかもしれませんが実は `(` を演算子と見ることで同じように処理することができます。`)` は `<式>` が終わったという目印になります。カンマを挟んで並んだ式を読むところは今回新たに作ります[^comma-operator]。

[^comma-operator]: カンマも演算子と見る設計も可能だと思いますがやっていません。1.2 の「コラム：タプルと配列」に関連。

`(` の優先度は高めです。`2 * add(3, 4)` であれば `add(3, 4)` してから `2` とかけ算してほしいですよね。`2 * add` を先に計算するというわけにはいきません。

現時点ではまだ意味のあるコードは書けませんが、`foo(2)(3, 4)` のような形も考えておきます。これは左結合で `(foo(2))(3, 4)` と解釈され、`foo(2)` が 2 引数の関数を返し、その関数に `(3, 4)` という引数を与えて呼び出すことになります。

`_call` を `_mul_div_mod` と `_primary` の間に挟みます。


```diff py

+        target = self._primary()
+        while self._current_token() == "(":
+            self._current_and_advance()

```

`_call()` は `_binary_left` と類似の構造ですがループの中身が違うので専用に書いています。「プライマリ式」を読んだら、次のトークンが `"("` である間 `_comma_separated_exprs(")")` で引数を読み、`")"` を読み飛ばしています。`add(2, 3)` や `foo(2)(3, 4)`、`add(add(2, 3), 4)` など思い浮かべながら追いかけてみてください。

`target = self._primary()` で `<関数>` にあたる部分を読んでいます。たいてい `add` のような名前だけど式でも OK ですよ、っていうところですね。`_primary()` では値を読むことが多いですが `()` に入れればどんな式でも書けますし左結合で関数呼び出しの結果を入れることもあります（`foo(2)(3, 4)` のようなとき）。なので `name` みたいな名前だとちょっと意味が狭いなということで `target` という名前をひねり出しました。いずれ関数以外も入る予定だったりするので。

カンマで区切られた「式」を読みます。`terminator` は終わりを表すトークンです。今は `)` 以外を扱うことはないので引数にしなくてもいいんですが後で別パターンが出てくるので先回りして [^not-yagni]。`cse` は Comma Separated Expression の略のつもりです[^maybe-not-in-fasion]。

[^not-yagni]: 必要かどうかわからないのに無駄に汎用的に作るなよ、っていう YAGNI（You Ain't Gonna Need It: 必要じゃないかもしれないよ）原則っていうのがあって、微妙に違反気味ですがあとで使うことは決定なので！

[^maybe-not-in-fasion]: 世間では変数名は略さず丁寧に、という傾向にあるようですが、局所的に使う変数名で、見ればわかってもらえるだろうと思ったら短くしちゃう派です。

実行例に進みます。

```py
    print("Function call:")

    print(toil.ast(r""" add(2, 3) """))  # -> ('add', [2, 3])
    print(toil.walk(r""" add(2, 3) """))  # -> 5

    print(toil.walk(r""" add(2 + 3, add(4, 5)) """))  # -> 14
    print(toil.walk(r""" add(2, 3) * 4 """)) # -> 20

    toil.walk(r""" myadd := add """)
    print(toil.walk(r""" myadd(2, 3) """))  # -> 5
```

`add` はただの変数だから `myadd := add` のようなことができますよ、というのは第 1 章でもやりました。
