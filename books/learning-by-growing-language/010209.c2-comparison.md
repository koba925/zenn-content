等号 `==` と比較演算子 `<` ・ `>` を実装します。AST では `("equal", ...)` や `("less", ...)`・`("greater", ...)` になるものです。`<=` や `>=` は 4 章で追加します。

現在の文字が `=` だったら 1 文字進めて文字を見て、今度も `=` だったらさらに 1 文字進めます。その後トークンを切り出すことで 2 文字の演算子として扱うことができます。まだ 1 文字の `=` が出てきていませんが、1 文字の `=` の場合も正しく扱えていることも見てください。

`<` ・ `>` は 1 文字の演算子なのでいつものパターンです。`<=` や `>=` も扱うようにするときは `<` ・ `>` を `=` と同じように扱います。

構文解析の修正は上で決めた規則を書くだけ。`_binary_left()` を作った効果で簡単に済んでいます。

```diff py

-    def _expression(self): return self._add_sub()
+    def _expression(self): return self._comparison()

+    def _comparison(self):
+        return self._binary_left({
+            "==": "equal", "<": "less", ">": "greater"
+        }, self._add_sub)
 
     def _add_sub(self):
         return self._binary_left({"+": "add", "-": "sub"}, self._mul_div_mod)
```

## コラム：等号・比較演算子の結合規則

> 結合は左結合とします。

遡ると C 言語の `==` が左結合です。もっと遡ってもそうかもしれない。なので、C 言語の影響を受けている言語の多くは `==` が左結合です。

で、なんで C 言語の `==` が左結合なのかというと・・・たぶん楽だからです。いや楽って言うからアレなんですけど、単純なしくみでちゃんと目的が果たせればそれでいいよねバグも出ないし、っていう感じなのではないかと。何も調べてませんけどたぶん。

とはいうもののわかりづらいというのは私も思います。別のアプローチをとっている言語もあります。

ひとつのアプローチは結合「しない」という作戦です。`2 == 2 == 2` も `2 == 2 == True` もエラー。これを実装するのは簡単で、`_binary_left()` の `while` を `if` にするだけです。メジャーな言語では Rust がそうです。厳しいところが Rust らしい。Go もそうです。C を作った人たちが作ってるのに・・・ずっと後悔してたんでしょうか・・・C に合わせてた人たちの立場は・・・

もうひとつのアプローチはご存じの Python。直観に合う答えを返してくれます。

```py
>>> 2 == 2 == 2
True
>>> 2 < 3 < 4
True
```
