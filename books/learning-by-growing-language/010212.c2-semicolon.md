順次実行を表すセミコロンを導入します。

C 言語系の文法では式の最後にセミコロンをつけると文になる、というのが基本ですが、Toil はすべてが式で文がないこともあり、セミコロンを演算子として実装します。

セミコロン演算子？と思われるかもしれませんが、以下のように考えると `+` と `;` は実は似たようなことをしていると納得できるでしょう。

`A + B` は `A` を評価して `B` を評価してから `A` の評価結果と `B` の評価結果を足して返す
`A ; B` は `A` を評価して `B` を評価してから `A` の評価結果を捨てて `B` の評価結果を返す

セミコロンの前のスペースを取って `A; B` と書けばなんとなく普通っぽくなります。ただ、普通のセミコロンと違って `<式>` と `<式>` の**間**にしかセミコロンが書けません。`A; B;` はエラーです。`2 + 3 +` がエラーになるのと同じです。 最初はエラーを出してしまうかもしれませんがしばらくすれば慣れます。たぶん。

Rust は `A; B` と書くと `A; return B;` と同じ意味になりますので見た目は似ています。コンパイラの中では違う考え方をしてそうな気がしますが（未確認）。

字句解析でセミコロンを通します。

優先度は最低です。`a := 2; b := 3` が `(a := 2); (b := 3)` と解釈されるようにです。`a := (2; b) := 3` では意味が分かりませんね。なので `_sequence` を `_expression` と `_define_assign` の間に挟みます。

左結合の二項演算子と思って構文解析してもいいんですけれどもせっかく `seq` は任意の数の式をまとめて扱えるので多項演算子として扱います。


```diff py
 class Parser:
     ...
 
 
-    def _expression(self): return self._define_assign()
+    def _expression(self): return self._sequence()
+
+    def _sequence(self):
+        exprs = [self._define_assign()]
+        while self._current_token() == ";":
+            self._current_and_advance()
+            exprs.append(self._define_assign())
+        return exprs[0] if len(exprs) == 1 else ("seq", exprs)
 
     def _define_assign(self):
         ...
```

セミコロンでつながった式が二つ以上あるときは `("seq", [<式>, <式>, ...])` に、式が一つだけの時は単に `<式>` にしたいんですが、`<式>` が複数あると思って最初の式を配列に入れてスタートし、ループを抜けてもそのまま配列の要素がひとつだけだったら中身を取り出して返すことによってそれを実現しています。

実行例。

```py
    print("Sequence:")

    print(toil.ast(r""" print(2); print(3) """))
    # -> ('seq', [('print', [2]), ('print', [3])])
    print(toil.walk(r""" print(2); print(3) """))  # -> 2\n3\nNone

    print(toil.ast(r""" 2 + 3; 4 + 5; 6 + 7 """))
    # -> ('seq', [('add', [2, 3]), ('add', [4, 5]), ('add', [6, 7])])
    print(toil.walk(r""" 2 + 3; 4 + 5; 6 + 7 """))  # -> 13

    print(toil.ast(r""" 2 + 3 """))
    # -> ('add', [2, 3])

    # toil.walk(r""" 2; """)  # -> Invalid token
    # toil.walk(r""" ;2 """)  # -> Invalid token
    # toil.walk(r""" 2;;3 """)  # -> Invalid token
```

注意するところはセミコロンがなければ `seq` を使わない AST になる、くらいでしょうか。
