title: "3.1 スタックマシン"
---

ということで、この章では中間コードへのコンパイラと仮想マシンを作っていきます。

コンパイラとか仮想マシンとか聞くとますます難しそうな気がするかもしれませんが、第 1 章・第 2 章同様ごくごく小さいちっぽけなところからやっていきますので大丈夫です。コンパイラや仮想マシンがややこしくならないよう、中間コードのレベルでかなりのことをやってくれるようにしています。

本書で作る仮想マシンは「スタックマシン」と呼ばれるものです。スタックマシンというアイデアはとてもおもしろく感じていて、本書で紹介するコンセプトの中でも上位に入ります[^other-ideas]。スタックマシンはスタックを作業場所として各種の処理を行います。

[^other-ideas]: ほかには評価器・クロージャ・再帰下降構文解析・マクロあたりがお気に入りです。

スタックというのはデータ構造の一種で、一連の値を覚えておくものですが、特徴はデータを取り出したいときは最後に入れたものしか取り出せないというところにあります。Python では配列をスタックとして用いることができますが、その時に使えるのが `append()` と `pop()` だけということです。こんな感じ。

```
初期状態 : []
append(2): [2]
append(3): [2, 3]
append(4): [2, 3, 4]
pop()    : [2, 3] → 4 が返る
pop()    : [2] → 3 が返る
pop()    : [] → 2が返る
```

こんな不自由なデータ構造がいかにうまく作業をこなすのに役立つか、これから少しずつ見ていきます。

スタックマシンでないものは何かというとレジスタマシンというのがあります。レジスタマシンは現実の CPU に近い方式で、一定数の「レジスタ」を作業場所として使います。レジスタマシンのほうがより高速な処理系になると言われていますが、それはそれでいろんな工夫を重ねたうえでの話。入門にはスタックマシンが向いています。

仮想マシンの最初の実装です。

```py
 class Evaluator:
     ...


+class VM:
+    def __init__(self, code):
+        self._code = code
+        self._ip = 0
+        self._stack = []
```

`VM` クラスを追加します。

`_code` はスタックマシンへの命令（Instruction）を覚えておく変数です。

命令は `(<オペコード>, <オペランド>, ...)` という形です。オペコード（Opcode）[^spelling]はその命令が何をするのかを表し、オペランド（Operand）は命令のための追加情報です。関数と引数みたいなものです。いわゆるバイトコードインタプリタでは、これらは普通ただのバイト（意味はいろいろあるとして）なんですが、Toil ではわかりやすさや書きやすさを優先して文字列やその他のデータ構造をそのまま使います。そのため、他の書籍や記事で使われる「オペコード」「オペランド」とはちょっとイメージが違うと思いますのでご注意ください。文字列やデータ構造も結局それを表しているのは数やアドレス（アドレスも結局は数）なので、そこまで考えれば実は根本的に違うというわけではありません。

[^spelling]: スペル注意です。オペコードと言いますが Opecode ではありません。

`_ip` は `_code` の中でどの命令を実行しているかを表すインデックスで、`_stack` はもちろんスタックです。どのように使うかはおいおいやっていきましょう。

次が仮想マシンのメインループです。

```diff py
 class VM:
     def __init__(self, code):
         ...

+    def execute(self):
+        while (inst := self._code[self._ip]) != ("halt",):
+            self._ip += 1
+            match inst:
+                case ("const", val): self._stack.append(val)
+                case _:
+                    assert False, f"Invalid instruction @ execute(): {inst}"
+        assert len(self._stack) == 1, f"Invalid stack state @ execute(): {self._stack}"
+        return self._stack.pop()
```

```py
        while (inst := self._code[self._ip]) != ("halt",):
```

`inst` に今見ている命令を保存し、`halt` が来るまでループします。Halt というのは止めるという意味の英語で、コンピュータの世界では OS を止めたり CPU を止めたりするときによく使われます。

```py
            self._ip += 1
```

命令を読んだらすぐさま `_ip` を進めるのが定石です。本物の CPU もそのように動きます。そういう意味では `_ip` は次の命令を指すインデックスととらえたほうがより正確です。

```py
            match inst:
                case ("const", val): self._stack.append(val)
```

命令が `("const", val)` の形だったら、スタックに `val` の値を積みます[^push-onto]。スタックマシンのお約束として、命令はその結果をスタックに積む、というのがあります。積まれた結果がスタックに残り、結果の値や次の命令の入力として使われます。

[^push-onto]: スタックは最後に入れた値しか取り出せないことから、重ねた皿などに例えられることが多く、「積む」という言い方を使います。

```py
                case _:
                    assert False, f"Invalid instruction @ execute(): {inst}"
```

想定していない命令だったらエラー。

ここまでをループします。

```py
        assert len(self._stack) == 1, f"Invalid stack state @ execute(): {self._stack}"
        return self._stack.pop()
```

`halt` に到達してループを抜けたとき、スタックに残っている値がプログラム全体の値です。1 個だけ残っているのが正しい状態で、0 個でも 2 個以上でもエラーです。

```diff py
 class Interpreter:
     ...

     def walk(self, src):
         ...
 
+    def execute(self, code):
+        return VM(code).execute()
```

`Interpreter` クラスに中間コードを受け取って仮想マシンで実行する `execute()` メソッドを追加します。

動かしてみましょう。

```py
    print("Stack machine:")

    print(toil.execute([('const', 2), ('halt',)])) # -> 2
    print(toil.execute([('const', None), ('halt',)])) # -> None
    print(toil.execute([('const', True), ('halt',)])) # -> True
    print(toil.execute([('const', False), ('halt',)])) # -> False

    # toil.execute([('halt',)]) # -> Invalid stack state
    # toil.execute([('const', 2), ('const', 3), ('halt',)]) # -> Invalid stack state
    # toil.execute([('not_op',), ('halt',)]) # -> Invalid instruction
```

第 1 章で言うとこのコードに相当します。AST を中間コードにコンパイルする部分がないのでまだ半分ですが。

```py
    print(toil.eval(None)) # -> None
    print(toil.eval(True)) # -> True
    print(toil.eval(False)) # -> False
    print(toil.eval(2)) # -> 2
```

スタックに 1 個も値がないときや 2 個以上あるとき、知らない命令が来たときはエラーです。
