title: "3.2 （中間コードへの）コンパイラ"

次は中間コードへのコンパイラです。AST を入力として、中間コードを出力します。AST は第 2 章の字句解析・構文解析器が出力するものをそのまま使います。

コンパイラはここから始めます。

```py
 class Evaluator:
     ...


+class Compiler:
+    def __init__(self, expr):
+        self._expr = expr
+        self._code = []
+
+    def compile(self):
+        self._expression(self._expr)
+        self._code.append(("halt",))
+        return self._code
+
+    def _expression(self, expr):
+        match expr:
+            case None | bool() | int(): self._code.append(("const", expr))
+            case _: assert False, f"Unsupported expression @ compile(): {expr}"


 class VM:
     ...
```

`Compiler` クラスを追加します。

`_expr` は受け取った AST を覚えておく変数、`_code` はコンパイルした命令を覚えておく変数で、`Compiler` クラスの出力になります。

`compile()` は受け取った AST をコンパイルして、最後に `halt` 命令を追加して返します。

`_expression()` は `_expr` が `None` や `bool` や `int` なら `const` 命令を出力します。`const` はスタックに値を積む命令でした。スタックの一番上に結果が残っているようにして終るのがスタックマシンのお約束でした。AST から `2` を読んだら `("const", 2)` を出力します。仮想マシンは `("const", 2)` を実行するとスタックに `2` を積みます。これが `2` という AST を実行した結果になる、というわけです。より複雑な計算で使う値は `const` でスタックに積んで使います。そういった例は後の節で。

それ以外ならエラーです。

仮想マシンの動きを思い出しながら読んでみてください。

`Interpreter` クラスに AST を受け取って中間コードにコンパイルする `compile()` メソッドと、ソースコードを受け取って中間コードを出力する `code()` メソッドを追加します。

```diff py
 class Interpreter:
     ...

     def walk(self, src):
         ...

+    def compile(self, ast):
+        return Compiler(ast).compile()
+
+    def code(self, src):
+        return self.compile(self.ast(src))

     def execute(self, code):
         ...
```

あと、コンパイル結果をちょっと見やすくする `print_code` 関数を追加しておきます。

```diff py
 if __name__ == "__main__":
     import sys
 
     toil = Interpreter()
 
+    def print_code(code):
+        for addr, inst in enumerate(code): print(f"{addr:3}: {inst}")

     ...
```

実行例です。コンパイルするだけにしておきます。

```py
    print("Compile:")

    print_code(toil.code(r""" 2 """))
    # ->   0: ('const', 2)
    # ->   1: ('halt',)

    # toil.compile((2, 3, 4)) # -> Unsupported expression
```

`None` でも `bool` でも `int` でもないものを渡すとエラーになることを確認しています。字句解析や構文解析は第 2 章のままですので、構文のエラーは確認済みとし、改めて試してはいません。

予想はついてると思いますが、これら仮想マシンとコンパイラを組み合わせると、最初の中間コード式インタプリタになります。進みましょう。
