title: "2.6 かけ算・割り算・剰余"
---

かけ算・割り算・剰余は `mul` ・ `div` ・ `mod` の式に直します。

`2 * 3` → `("mul", [2, 3])`
`6/2` → `("div",[6, 2])`
`7 % 3` → `("mod",[7, 3])`

今回考慮しないといけないのは `2 + 3 * 4` のように足し算・引き算と、かけ算・割り算・剰余が混ざった計算ではかけ算・割り算・剰余のほうを先に行う必要があるということです。AST では `("add", [2, ("mul", [3, 4])])` のようになっていないといけません。これを、かけ算・割り算・剰余は足し算・引き算よりも優先順位が高い、と言います。今後の演算子の追加でも毎回優先順位を決めていきます。

これをどうやって実現するかですが、「かけ算・割り算・剰余」を含めたルールを以下のように定めます。

「プログラム」は「式」、
「式」は「足し算・引き算」、
「足し算・引き算」は「かけ算・割り算・剰余」のあとに（`+` または `-` ＋「かけ算・割り算・剰余」）が 0 個以上続いたもの、
「かけ算・割り算・剰余」は「プライマリ式」のあとに（`*` または `/` または `%` ＋「プライマリ式」）が 0 個以上続いたもの、
「プライマリ式」は「数」

これで「かけ算・割り算・剰余」の優先順位が「足し算・引き算」よりも上になりました。

って、ええ？

いやまあ、まだコード書いてないんで「なりました」ってことはないんですがあとはこれをコードにするだけで終わりです。

「足し算・引き算」は「かけ算・割り算・剰余」に「かけ算・割り算・剰余」を足したり引いたりしたもの、と読めば「かけ算・割り算・剰余」を先にやって、そのあと足したり引いたりするんだな、という気がするでしょう（たぶん）。

`2 + 3 * 4` で考えると、`2` という（`*` や `/` や `%` が 0 個の）「かけ算・割り算・剰余」と、`3 * 4` という「かけ算・割り算・剰余」を足したものですので、カッコをつけると `(2) + (3 * 4)` という意味になり、`3 * 4` が先に計算されることになります。

コード見てみましょう。

```diff py
 class Scanner:
     ...
     def tokenize(self):
         while True:
             ...
             match self._current_char():
                 ...
                 case c if c.isdecimal(): self._number()
-                case c if c in "+-":
+                case c if c in "+-*/%":
                     self._tokens.append(c); self._advance()
                 ...
```

字句解析が `*`・`/`・`%` のトークンを許すようにします。

次は、「足し算・引き算」は・・・と「かけ算・割り算・剰余」は・・・を書きます。考え方が同じなのでコードもそっくりです。

```diff py
 class Parser:
     ...

     def _add_sub(self):
         ops = {"+": "add", "-": "sub"}
-        left = self._primary()
+        left = self._mul_div_mod()
         while type(op := self._current_token()) is str and op in ops:
             self._current_and_advance()
-            left = (ops[op], [left, self._primary()])
+            left = (ops[op], [left, self._mul_div_mod()])
         return left

+    def _mul_div_mod(self):
+        ops = {"*": "mul", "/": "div", "%": "mod"}
+        left = self._primary()
+        while type(op := self._current_token()) is str and op in ops:
+            self._current_and_advance()
+            left = (ops[op], [left, self._primary()])
+        return left
 
     def _primary(self):
         ...
```

そっくりだな、わかった、と思ったらもう一歩進みましょう。そっくりすぎるので。Don't Repeat Yourself（DRY）です。短く言うと「同じことを何度も書かずに関数にするとかして共通化しろ」ってことなんですが見た目はそっくりだけど本質的には異なるものを共通化しちゃうとかえって困ったことになるので注意が必要です。なんだってそうですが。`_add_sub()` と `_mul_div_mod()` はどちらも左結合の二項演算子という点で本質的な共通点があるので共通化して大丈夫です。というかするべきです。

ふたつのメソッドを見比べると、`{"+": "add", "-": "sub"}` と `{"*": "mul", "/": "div", "%": "mod"}`、`_mul_div_mod` と `_primary` が違っているだけであとは全く同じです。そこで、違っているところを引数にしたこんな関数を作ってみましょう。Binary というのは二項演算子（Binary Operator）からとっています。`+`・`-`・`*`・`/` などはみな二項演算子で、これからも出てきます。

```py
     def _primary(self):
         ...
 
+    def _binary_left(self, ops, sub_elem):
+        left = sub_elem()
+        while type(op := self._current_token()) is str and op in ops:
+            self._current_and_advance()
+            left = (ops[op], [left, sub_elem()])
+        return left
```

すると、`_add_sub()` と `_mul_div_mod()` は `_binary_left()` を呼ぶだけで済んでしまいます。

```py
    def _add_sub(self):
        return self._binary_left({"+": "add", "-": "sub"}, self._mul_div_mod)

    def _mul_div_mod(self):
        return self._binary_left({
            "*": "mul", "/": "div", "%": "mod"
        }, self._primary)
```

`_binary_left()` の中の `ops` を `{"+": "add", "-": "sub"}` に入れ替え、`sub_elem` を `self._mul_div_mod` に入れ替える [^bound-method] と上にあげた `_add_sub()` に戻ることを確認してください。`_mul_div_mod()` も同様ですし、今後出てくる「左結合の二項演算子」も同じように書けます。行数が減るというのも効果ですが、これは左結合の二項演算子である、ということがコード上に表現されたということが大事です。

[^bound-method]: ここで、`sub_elem` に `self._mul_div_mod` を渡しているのが気になる人もいるでしょう。このように `self.<メソッド>` という形にすると、数や文字列など他の値と同じように引数として渡したり関数から返したりすることができます。関数自体が値であるのと同じです。束縛メソッド（Bound method）と言います。`self` とくっついたメソッド、という意味です（bound という言葉にはもう少し深みがあります）。

実行例です。

```py
    print("Multiplication, division, and modulo:")

    print(toil.ast(r""" 2 * 3 """))  # -> ('mul', [2, 3])
    print(toil.walk(r""" 2 * 3 """))  # -> 6

    print(toil.ast(r""" 6 / 2 """))  # -> ('div', [6, 2])
    print(toil.walk(r""" 6 / 2 """))  # -> 3

    print(toil.ast(r""" 7 % 3 """))  # -> ('mod', [7, 3])
    print(toil.walk(r""" 7 % 3 """))  # -> 1

    print(toil.walk(r""" 2 * 3 * 4 """))  # -> 24
    print(toil.walk(r""" 24 / 4 / 2 """))  # -> 3
    print(toil.walk(r""" 4 * 3 / 2 """))  # -> 6

    print(toil.ast(r""" 2 + 3 * 4 """))  # -> ('add', [2, ('mul', [3, 4])])
    print(toil.walk(r""" 2 + 3 * 4 """))  # -> 14
    print(toil.walk(r""" 2 * 3 + 4 """))  # -> 10
```

左結合になっていたりかけ算が優先されていたりすることを確認してください。
