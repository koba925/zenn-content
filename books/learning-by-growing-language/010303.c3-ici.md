---
title: "3.3 中間コード式インタプリタ"
---

ソースコードを中間コードにコンパイルするコンパイラと、中間コードを読んで実行する仮想マシンができました。これらを組み合わせて、ソースコードを実行できるようにします。これで中間コード式インタプリタの骨組みができ、今後機能を追加していく土台となります。

本質的な変更はこれだけです。まさしくコンパイラの出力（`code()`）と仮想マシン（`execute()`）をつなげただけですね。

```diff py
 class Interpreter:
     ...

     def execute(self, code):
         ...
 
+    def run(self, src):
+        return self.execute(self.code(src))

 
 if __name__ == "__main__":

     ...
```

ツリーウォーク式のインタプリタを実行するときに `walk()` にしてたのは `run()` をここで使いたかったからです。中間コード式インタプリタのほうがより高速に動くことになってますしね[^ambitious-plan]。

[^ambitious-plan]: ツリーウォーク式インタプリタの処理と中間コード式インタプリタの処理を同居させておく必要は必ずしもない（というか同居してないほうが普通）なくて、片方だけにしておくならこんな使い分けを考えるのも不要なんですが、そこは遠大な計画がありまして。

以下も同居した処理系を両方動かすための修正です。

```diff py
 if __name__ == "__main__":
     import sys
 
     toil = Interpreter()
 
     def print_code(code):
         for addr, inst in enumerate(code): print(f"{addr:3}: {inst}")
 
-    def repl():
+    def repl(walk_or_run):
         while True:
             print("\nInput source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):")
             if (src := sys.stdin.read()) == "":
                 exit(0)
             try:
                 expr = toil.ast(src)
                 print("AST:", expr, sep="\n")
-                print("Output:")
-                result = toil.eval(expr)
+                if walk_or_run == "walk":
+                    print("Output:")
+                    result = toil.eval(expr)
+                else:
+                    code = toil.code(src)
+                    print("Code:")
+                    print_code(code)
+                    print("Output:")
+                    result = toil.execute(code)
                 print("Result:", result, sep="\n")
             except AssertionError as e:
                 print("Error:", e, sep="\n")
 
-    def from_file(filename):
-        with open(filename, "r") as f: result = toil.walk(f.read())
+    def from_file(walk_or_run, filename):
+        with open(filename, "r") as f:
+            if walk_or_run == "walk":
+                result = toil.walk(f.read())
+            else:
+                result = toil.run(f.read())
         exit(result if isinstance(result, int) else 255)
 
     match sys.argv:
         case [_]: pass
-        case [_, "--repl"]: repl()
-        case [_, filename]: from_file(filename)
+        case [_, "--repl"]: repl("walk")
+        case [_, "--rcepl"]: repl("run")
+        case [_, "--walk", filename]: from_file("walk", filename)
+        case [_, "--run", filename]: from_file("run", filename)
         case _: assert False, f"Invalid command line: {sys.argv}"
 
     # Example     ...
```

`--repl` オプションはツリーウォーク式の REPL を起動します。
`--rcepl` オプションは中間コード式の REPL を起動します。`rcepl` は Read-Compile-Execute-Print-Loop の頭文字語で、造語です。ヨソで使っても通じません。
`--walk <ファイル名>` または `--run <ファイル名>` はそれぞれツリーウォーク式、中間コード式で指定したファイルを実行します。

あとはツリーウォーク式か中間コード式かで `Interpreter` のメソッドの呼び方を変えていることと、中間コード式の REPL（RCEPL）では中間コードも表示しているくらいです。

実行例です。まずは RCEPL から。

```
$ ./toil.py --rcepl

Input source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):
2
AST:
2
Code:
  0: ('const', 2)
  1: ('halt',)
Output:
Result:
2

Input source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):
print(2)
AST:
('print', [2])
Error:
Unsupported expression @ compile(): ('print', [2])

Input source and enter Ctrl+D (Linux/Mac) or Ctrl+Z (Windows):

```

ファイルから実行します。`2` とだけ書いた `2.toil` というファイルを用意してください。

```
$ ./toil.py --run 2.toil
$ echo $?
2
```

できました。あとは機能を追加していくだけです！
