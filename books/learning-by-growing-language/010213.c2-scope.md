---
title: "2.13 scope-end"
---

今までは演算子や関数呼び出しなど、構文ではあるもののあんまり構文っぽくないものを導入していました。ここからは見た目も構文らしいものを導入していきます。といってもこれまでとそれほど違うことをするわけではありません。「構文」と言ってもやはり「式」ではありますのでそこはご注意ください。

最初に導入するのは `scope <式> end` です。これはスコープを作って `<式>` をその中で評価するもので、AST では `("scope", [<式>])` となります。

あたらしいトークンはないので字句解析には手を入れる必要はありません[^required-if-keyword]。

[^required-if-keyword]: `scope` や `end` などをキーワードとして特別扱いするようになればそれらを追加する必要がありますが Toil では行いません。

優先度や結合順をどう考えるかですが、Toil では `scope <式> end` をプライマリ式として扱うことにします。ひらたく言えば、`scope <式> end` をカッコ `(<式>)` と同類と見る、ということです。文字数が多いだけで `<式>` を前後から挟んでいることには違いないですね。どこまでつながっているかは `end` で明確になっているので結合順は考える必要がありません。

```diff py
 class Parser:
     ...
 
     def _primary(self):
         match self._current_token():
             ...
             case "(": return self._group()
+            case "scope": return self._scope()
             case str(name) if is_ident(name): return self._current_and_advance()
             ...
 
     def _group(self):
         self._current_and_advance()
         expr = self._expression()
         self._consume(")")
         return expr
 
+    def _scope(self):
+        self._current_and_advance()
+        body_expr = self._expression()
+        self._consume("end")
+        return ("scope", [body_expr])
 
     def _binary_left(self, ops, sub_elem):
         ...
```

`scope` を見つけたらトークンをひとつ進めて `<式>` を読み、`end` が来ていることを確認して `scope` の AST を返します。

カッコの処理（`_group()`）と見比べると、構造が全く同じであることがわかるでしょう。

実行例です。

```py
    print("Scope:")

    print(toil.ast(r""" a := 2; scope a end """))
    # -> ('seq', [('define', ['a', 2]), ('scope', ['a'])])

    print(toil.walk(r""" a := 2; scope a end """)) # -> 2
    print(toil.walk(r""" a := 2; scope scope a end end """)) # -> 2

    print(toil.walk(r""" a := 2; scope a := 3 end """)) # -> 3
    print(toil.walk(r""" a """)) # -> 2

    print(toil.walk(r""" a := 2; scope a = 3 end """)) # -> 3
    print(toil.walk(r""" a """)) # -> 3

    # toil.walk(r""" a := 2; scope d = 3 end """)  # -> Undefined variable
    # toil.walk(r""" scope 2 """)  # -> Expected end
    # toil.walk(r""" scope end """)  # -> Expected end
```

スコープができていることを確認してください。

これが Toil における構文の導入の基本パターンとなります。同じようにして `if`・`while` などを導入していきます。
