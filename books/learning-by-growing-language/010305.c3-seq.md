電卓までなら前節の仮想マシン＋αくらいでできるのですが、プログラミング言語にしようとなるとまだ経路の違う命令が必要になってきます。

まずは `seq`（順次実行）を実行できるようにします。

仮想マシンに `pop` 命令を追加します。

```diff py
 class VM:
     ...
 
     def execute(self):
         while (inst := self._code[self._ip]) != ("halt",):
             self._ip += 1
             match inst:
                 case ("const", val): self._stack.append(val)
+                case ("pop",): self._stack.pop()
                 case ("print",):
                     print(self._stack.pop()); self._stack.append(None)
                 ...
             ...
```

`("pop",)` という命令が来たら `self._stack.pop()` でスタックの最後の値を取り出します。取り出した値はなにも使わないので、最後の値を捨てているだけです。

こんな命令をどう使うかはコンパイラのほうで見てみましょう。

```diff py
class Compiler:
     ...
 
     def _expression(self, expr):
         match expr:
             case None | bool() | int(): self._emit("const", expr)
+            case ("seq", exprs): self._seq(exprs)
             case (op, [expr]):
                 ...
             ...

+    def _seq(self, exprs):
+        assert len(exprs) > 0, f"Empty sequence @ compile(): {exprs}"
+        for expr in exprs[:-1]:
+            self._expression(expr)
+            self._emit("pop")
+        self._expression(exprs[-1])

     def _emit(self, *inst):
         ...
```

`for expr in exprs[:-1]` は `exprs` に含まれる式の最後のひとつ以外についてループします。すくなくともひとつは式が含まれている必要があるので手前の `assert` でチェックしています。


`self._expression(expr)` では、`expr` を実行して、結果をスタックに残すコードを出力するのでした。ループの中にある `self._expression(expr)` がスタックに残した結果は直後に出力される `pop` で捨ててしまいます。

ループを抜けたら最後の式 `exprs[-1]` を実行します。この結果はそのままにしておき `seq` の値として使われることになります。

`Evaluator` の `_seq` や、構文解析でセミコロンを導入したときの「`A ; B` は `A` を評価して `B` を評価してから `A` の評価結果を捨てて `B` の評価結果を返す」と同じようなことをしています。

```py
    def _seq(self, exprs):
        val = None
        for expr in exprs: val = self.eval(expr)
        return val
```

実行例です。

```py
    print("Sequence:")

    print(toil.ast(r""" 2; 3; 4 """))
    # -> ('seq', [2, 3, 4])
    print_code(toil.code(r""" 2; 3; 4 """))
    # ->   0: ('const', 2)
    # ->   1: ('pop',)
    # ->   2: ('const', 3)
    # ->   3: ('pop',)
    # ->   4: ('const', 4)
    # ->   5: ('halt',)
    print(toil.run(r""" 2; 3; 4 """)) # -> 4
```

せっかく積んだ `2` や `3` を捨てて、`4` が結果になっています。

```py
    print(toil.ast(r""" print(2); print(3) """))
    # -> ('seq', [('print', [2]), ('print', [3])])
    print_code(toil.code(r""" print(2); print(3) """))
    # ->   0: ('const', 2)
    # ->   1: ('print',)
    # ->   2: ('pop',)
    # ->   3: ('const', 3)
    # ->   4: ('print',)
    # ->   5: ('halt',)
    print(toil.run(r""" print(2); print(3) """)) # -> 2\n3\nNone
```

`print` も `None` を返しますので途中の結果は捨てます。表示結果を見ると、`print(2)` も実行されていることが確認できます。

```py
    # toil.compile(("seq", [])) # -> Empty sequence
```


`("seq", [])` は `eval()` だと評価に成功して `None` が値となっていましたが、`execute()` ではエラーとしています。`_seq()` で `exprs` が `[]` であれば `None` を積むようにすれば `eval()` と互換にできますが、セミコロンをどう使ってもそういう AST は作れないのでこれでヨシとしています[^compatibility]。

[^compatibility]: ひとつの言語で複数の実装があれば互換性があるのが理想です。この実装では、Toil として書ける範囲では互換だけれども、中まで見ると `eval()` と `compile()`+`execute()` では動作がちょっと違っている、というギリギリで互換性が保たれている状態です。
