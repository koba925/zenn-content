title: "2.7 カッコ"
---

カッコです。カッコの働きは標準で決まっている結合順や優先順を無視して、カッコが囲んでいる式を先に計算する、ということですね。

カッコを導入するためにあらたな AST を用意する必要はありません。たとえば `(2 + 3) * 4` と `2 + 3` を先に計算したいなら `("mul", [("add", [2, 3]), 4])` と書けばよいからです。

カッコに入った式は以下のように扱います。

「プログラム」は「式」、
・・・
「プライマリ式」は「数」または `(`＋「式」＋`)`

驚きがパターン化してますがこれだけです！あとはコードを書くだけ。

```diff py
                 ...
                 case c if c.isdecimal(): self._number()
-                case c if c in "+-*/%":
+                case c if c in "+-*/%()":
                     self._tokens.append(c); self._advance()
                 ...
```

字句解析が `(`・`)` のトークンを許すようにします。もうここは大丈夫ですね。

次がミソで、再帰下降の「再帰」がよくあらわれた部分です。

```diff py
 class Parser:
     ...

     def _primary(self):
         match self._current_token():
             case int(): return self._current_and_advance()
+            case "(": return self._group()
             case invalid:
                 assert False, f"Invalid token @ _primary(): {invalid}"
 
+    def _group(self):
+        self._current_and_advance()
+        expr = self._expression()
+        self._consume(")")
+        return expr

     def _binary_left(self, ops, sub_elem):
         ...
```

`(` というトークンに出会ったら次のトークンに進み、「式」をパースして、`)` で閉じていることを確かめます。「プライマリ式」は「数」または `(`＋「式」＋`)`、のとおりです。

ごく素直にカッコに囲まれた「式」を評価する、と書いただけですが、これでどれだけ複雑な入れ子になったカッコも処理できてしまいます。すごくは見えないのにすごいというところがすごいです（語彙）。再帰なしで書こうと思ったらまあまあ大変です。しらんけど。

`)` で閉じていることは `_consume()` で確かめます（consume は「消費する」という意味）。ここにはこのトークンが来ているはず、というチェックは今後もたくさん出てきます。すべて `_consume()` で確かめます。

```diff py
     def _binary_left(self, ops, sub_elem):
         ...
 
+    def _consume(self, expected):
+        assert self._current_token() == expected, \
+            f"Expected {expected} @ _consume(): {self._current_token()}"
+        return self._current_and_advance()

     def _current_token(self): return self._tokens[self._pos]
         ...
```

やってることはただ期待しているトークンかどうか確認して、次のトークンに進んでいるだけです。

実行例です。

```py
    print("Paren:")

    print(toil.ast(r""" (2 + 3) * 4 """))  # -> ('mul', [('add', [2, 3]), 4])
    print(toil.walk(r""" (2 + 3) * 4 """))  # -> 20
    print(toil.ast(r""" 2 * (3 + 4) """))  # -> ('mul', [2, ('add', [3, 4])])
    print(toil.walk(r""" 2 * (3 + 4) """))  # -> 14

    print(toil.walk(r""" (2 + 3) """))  # -> 5
    print(toil.walk(r""" (2) """))  # -> 2
    print(toil.walk(r""" (5 - (4 - 2)) * 2 """))  # -> 6
    print(toil.walk(r""" (2 + 3) * (4 + 5) """))  # -> 45

    # print(toil.walk(r""" 2 + 3) """))  # -> Extra token
    # print(toil.walk(r""" (+) """))  # -> Invalid token
    # print(toil.walk(r""" (2 + 3 """))  # -> Expected )
    # print(toil.walk(r""" () """))  # -> Invalid token
```

できてますね。単なる `2` も「式」なのでカッコで囲んだ `(2)` も正しい「式」です。

これで四則演算＋カッコの計算ができるようになりました。すでに再帰下降構文解析の基礎的なところはマスターしています。
