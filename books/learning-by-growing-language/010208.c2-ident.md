次は、`None` と `bool` と識別子をやりましょう。要は数字じゃない単語をまとめてやっちゃおうということです。

「プライマリ式」は
「数」または `None` または `True` または `False` または
「識別子」または
　`(`＋「式」＋`)`

「識別子」は英字または `_` のあとに 0 個以上の英数字または `_` が続いたもの

と決めます。「識別子」の決め方はやはり最初の文字だけでそれが何かわかる、という考え方の例です。最初の文字が英字または `_` なら識別子ですねと。

今回は字句解析のほうが主役です。

ソースの冒頭に、文字の種類を判別するためのユーティリティ関数を追加します。

```diff py
+def is_ident_first(c): return c.isalpha() or c == "_"
+def is_ident_rest(c): return c.isalnum() or c == "_"
+def is_ident(s): return is_ident_first(s[0])
```

`is_ident_first(c)` は文字 `c` が識別子の最初の文字に該当するか、`is_ident_rest(c)` は `c` が識別子の 2 文字目以降の文字に該当するか、`is_ident(s)` は文字列 `s` が識別子であるかどうかを判断しています。`is_ident(s)` で最初の文字しか確認してないのはただの手抜きです。

字句解析の本体です。

```diff py
 class Scanner:
     ...

     def tokenize(self):
         while True:
             ...
             match self._current_char():
                 ...
                 case c if c.isdecimal(): self._number()
+                case c if is_ident_first(c): self._ident()
                 case c if c in "+-*/%()":
                    ...

     def _number(self):
         ...
 
+    def _ident(self):
+        self._advance()
+        while is_ident_rest(self._current_char()): self._advance()
+        match self._lexeme():
+            case "None": self._tokens.append(None)
+            case "True": self._tokens.append(True)
+            case "False": self._tokens.append(False)
+            case ident: self._tokens.append(ident)

     def _lexeme(self):
         ...
```

最初の文字が `is_ident_first()` だったら識別子と判断して `_ident()` に後の処理を任せます。

`_ident()` では、現在位置の文字が `is_ident_rest()` である間現在位置を進めます。`_lexeme()` で識別子を切り出して、`"None"`（これはまだ文字列）だったら `None`（これは本当に `None`）を、`"True"` だったら `True` を、`"False"` だったら `False` を `_tokens` に追加します。

どれでもなかったら識別子としてそのまま `_tokens` に追加します。AST では変数名や `if` などを文字列で表していたことを思い出してください。

字句解析ができたら構文解析のほうは大したことはしません。

```diff py
 class Parser:
     ...

     def _primary(self):
         match self._current_token():
-            case int(): return self._current_and_advance()
+            case None | bool() | int(): return self._current_and_advance()
             case "(": return self._group()
+            case str(name) if is_ident(name): return self._current_and_advance()
             case invalid:
                 assert False, f"Invalid token @ _primary(): {invalid}"
 
     ...
```

トークンが `None` や `bool` のときも、`int` の時と同じようにトークンをそのまま返し、現在位置を進めます。トークンが `str` で、名前が `is_ident` ならばやはりトークンをそのまま返し、現在位置を進めます。

今回の実行例はあまり面白味がありません。`None` や `True` や `False` があってもできることがあまりないので。識別子、つまり変数に至っては変数定義[^builtin]もないのでエラーを出して確認しています。
