title: "2.5 足し算と引き算"
---

字句解析・構文解析が足し算・引き算を処理できるようにします。具体的には、たとえば `"2 + 3"` を `("add", [2, 3])` に、`"9 - 4"` を `("sub", [9, 4])` に変換したりするということです。

ひとつ足し算または引き算しただけで終わりではなく、`+` や `-` でつながっていればどんどん見ていきます。このときに大事なのが結合順です。

`2 + 3 - 4` はまず `2 + 3` を計算してから `4` を引きますね。`3 - 4` を計算して、それを `2` からひくのではありません。カッコをつけて書けば `(2 + 3) - 4` という意味です。AST では `("sub", [("add", [2, 3]), 4])` となります。`2 + 3` を先に計算しているのがわかりますか？

左から計算していくので、これを左結合と言います。左側にある `2 + 3` の `+` のほうが右側にある `3 - 4` の `-` よりも結合力が強いので左の `+` のほうが先に `2` と `3` を結合する、って感じです。 

コードの説明から行きます。

```diff py
 class Scanner:
     ...
     def tokenize(self):
         while True:
             ...
             match self._current_char():
                 ...
                 case c if c.isdecimal(): self._number()
+                case c if c in "+-":
+                    self._tokens.append(c); self._advance()
                 case invalid:
                     assert False, f"Invalid character @ tokenize(): {invalid}"
             ...
```

字句解析が `+` と `-` をひとつのトークンとして扱うようにします。

`c in "+-"` は `c` が `"+"` か `"-"` のどちらかであれば真になります。`c in ["+", "-"]` と書いても同じ意味ですが短く書きました。`self._tokens.append(c)` ではその 1 文字だけをトークンとして `_tokens` に追加します。

```diff py
 class Parser:
     ...

     def parse(self):
         ...

-    def _expression(self): return self._primary()
+    def _expression(self): return self._add_sub()
+
+    def _add_sub(self):
+        ops = {"+": "add", "-": "sub"}
+        left = self._primary()
+        while type(op := self._current_token()) is str and op in ops:
+            self._current_and_advance()
+            left = (ops[op], [left, self._primary()])
+        return left

     def _primary(self):
         ...
```

構文解析を導入したとき、

`<プログラム>` は `<式>`、
`<式>` は `<プライマリ式>`、
`<プライマリ式>` は `<数>`

と書きました。ここでは以下のように `<足し算・引き算>` を加えます。

`<プログラム>` は `<式>`、
`<式>` は `<足し算・引き算>`、
`<足し算・引き算>` は `<プライマリ式>` のあとに（`+` または `-` ＋`<プライマリ式>`）が 0 個以上続いたもの、
`<プライマリ式>` は `<数>`

`<足し算・引き算>` のところは、`2 + 3 - 4` で言えば `2` のあとに `+ 3` が続き、さらに `- 4` が続いています。そういうことを言っています。

「0 個以上」というところにも注意してください。「`+` または `-`＋`<プライマリ式>`」が一つも続いてなくても、つまり単なる `2` も `<足し算・引き算>` の仲間だということです。1 個もない、と特別扱いするより 0 個ある、と考えたほうがうまくいきやすいのは法則です。

ここはけっこう密度が高いのでさらに分けて説明します。読んでわかる人は飛ばしてください。

```py
    def _expression(self): return self._add_sub()
```

`<式>` は `<足し算・引き算>`、をそのまま書きました。ここから下も「そのまま書いた」感じに読めれば 100 点なんですが、慣れるまではそう思えなくても普通です。

```py
    def _add_sub(self):
        ops = {"+": "add", "-": "sub"}
```

これは、`+` を `add` の式に、 `-` を `sub` の式に変換するための仕込みです。

```py
        left = self._primary()
```

ここから本体です。`2 + 3 - 4` を思い浮かべて読んでください。

`<プライマリ式>` のあとに、の `<プライマリ式>` をまず読みます。`2 + 3 - 4` なのでまずは `2` が読まれて `left` が `2` になります。

```py
        while type(op := self._current_token()) is str and op in ops:
```

続く `while` は、一言でいうと次のトークンが `+` か `-` の間繰り返す、という意味なんですが 1 行にいろいろ詰め込まれているので分解して説明します。

`op := self._current_token()` は現在のトークンを `op` に代入します。これは `=` と違って式であり、その値は `self._current_token()` の値となります。`2 + 3 - 4` で `2` を読んだ直後だと `op` や式全体の値は `"+"` になります。

`op in ops` の `in` は `tokenize()` にも出てきた `in` ですが、`ops` のような辞書に対して使うと、辞書のキーの中に `op` と一致するものがあるかを返します。`op` が `"+"` なら `True` になります。

`type(op := ...) is str and op in ops` は、`op` の型が `str` かどうかを確かめ、その時に限り `op in ops` の判定を行います。なぜあらかじめチェックしてから `op in ops` しているかというと、`op` が辞書のキーとして不適切な型だとエラーになってしまうからです。`op` が `"+"` や `"-"` でないときは端からお呼びでないので、まず `str` かどうか確認してからチェックしている、というわけです。

今は `op` が `"+"` なので `while` の条件が満たされ、`while` の本体が実行されます。

・・・「次のトークンが `+` か `-` の間繰り返す」と読めたでしょうか？健闘を祈ります。

```py
            self._current_and_advance()
```

まだ現在位置は `+` か `-` を指しているので、トークンをひとつ進めます

```py
            left = (ops[op], [left, self._primary()])
```

`ops` が `{"+": "add", "-": "sub"}`、`op` が `+` なので `ops[op]` は `"add"` になります。`left` は `2` でしたね。`self._primary()` でプライマリ式を読むと `3` が返ってきます。そうすると右辺は `("add", [2, 3])` と、`add` の式の形になります。これをあらためて `left` の値として `while` の条件判断に戻ります。

ここがちょっと面白いところで、2 周目も同様に考えると `op` が `-`、`left` が `("add", [2, 3])` で `self._primary()` が `4` となるので、`(ops[op], [left, self._primary()])` は `("sub", [("add", [2, 3]), 4])` となり、これが新しい `left` の値になります。こうしてループを回るごとに `left` が大きくなっていきます。

次のトークンは `"$EOF"` で、`while` の条件が満たされずループを抜けて、

```py
        return left
```

`left` つまり `("sub", [("add", [2, 3]), 4])` を返します。

できました。

なんだかいつの間に？という感じかもしれませんが、大事な処理ですので読み返したりデバッガで変数を見ながらステップ実行したりしながら時間をかけて理解してください。初めてならすぐにはわからなくても普通です。


参考までに、あんまりカッコつけずに書くとこれでも同じ動きです。見比べると上のコードの意味も見えてくるかもしれません。`while True:` と `break` を組み合わせるとか、`"add"` と `"sub"` を使い分けるためだけに `if` を使うとかがちょっと今ひとつに感じるので上のコードのように書いています。`+` と `-` のように 2 個の演算子で済まないときもあって `if` が増えてしまいますし。あと次節でやりたいこととの関連も。

```py
    def _add_sub(self):
        left = self._primary()
        while True:
            op = self._current_token()
            if op != "+" and op != "-": break
            self._current_and_advance()
            if op == "+":
                left = ("add", [left, self._primary()])
            else:
                left = ("sub", [left, self._primary()])
        return left
```

実行例では AST も出力するようにしました。計算結果だけでなく正しい AST ができていることも見ておいてください。

```py
    print("Addition and subtraction:")

    print(toil.ast(r""" 2+3 """))  # -> ('add', [2, 3])
    print(toil.walk(r""" 2+3 """))  # -> 5

    print(toil.ast(r""" 5 - 3 """))  # -> ('sub', [5, 3])
    print(toil.walk(r""" 5 - 3 """))  # -> 2

    print(toil.ast(r""" 2 + 3 + 4 """))  # -> ('add', [('add', [2, 3]), 4])
    print(toil.walk(r""" 2 + 3 + 4 """))  # -> 9

    print(toil.ast(r""" 9 - 4 - 3 """))  # -> ('sub', [('sub', [9, 4]), 3])
    print(toil.walk(r""" 9 - 4 - 3 """))  # -> 2

    print(toil.ast(r""" 2 + 3 - 4 """))  # -> ('sub', [('add', [2, 3]), 4])
    print(toil.walk(r""" 2 + 3 - 4 """))  # -> 1

    # print(toil.walk(r""" 2 + """))  # -> Invalid token
    # print(toil.walk(r""" 2 - + 3 """))  # -> Invalid token
    # print(toil.walk(r""" -2 """))  # -> Invalid token
```

`-2` という一見正しそうな表記がエラーになっていることに注意してください。引き算は実装しましたが、数字をマイナスにするための `-` は実装していませんので、プライマリ式を読もうと思ったら変なの来たよ！としか思われません。自分で実装しないと、いつの間にかできるようになってたりはしません。
